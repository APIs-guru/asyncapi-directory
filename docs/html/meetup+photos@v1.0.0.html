Name Default value Possible values Description
photo_id None Any

Photo IDs, separated by commas
group_urlname None Any

Group urlnames, separated by commas
event_id None Any

Event ids, separated by commas. These may contain alphanumeric autoscheduled event ids, only photos of reified events will be returned
tagged None Any

Tagged with members with these IDs, separated by commas
photo_album_id None Any

Photo Album IDs, separated by commas
member_id None Any

Uploaded by members with these IDs, separated by commas
time None Any

Return photos uploaded within the given time range, defined by two times separated with a single comma. Each end of the range may be specified with relative dates, such as "1m" for one month from now, or by absolute time in milliseconds since the epoch. If an endpoint is omitted, the range is unbounded on that end. The default value is unbounded on both ends (though restricted to the search window described above).
fields None Any

comma-delimited optional response properties such as member_country, member_city, member_state, and self
group_id None Any

Group IDs, separated by commas
callback None Any

Name of a function to be called with an array of photo notification objects. If this parameter is not supplied, the chunked stream is joined instead.
since_mtime None Any

Should be supplied for all but the first polling request, so that any missed notifications are can be sent in an immediate response
since_count None Any

Request that some number of recent messages be sent immediately, if available. May not be specified in the same request as since_mtime.
